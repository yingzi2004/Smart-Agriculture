# 智慧农业 Demo 开发骨架说明

## 当前架构

当前项目是 RuoYi 单体结构，不是前后端分离：

```text
浏览器
  -> Spring Boot 8080
  -> Thymeleaf 首页菜单
  -> static/agriculture 静态页面
  -> /agriculture/api/* 后端 JSON 接口
```

## 前端入口

智慧农业前端页面在：

```text
ruoyi-admin/src/main/resources/static/agriculture/
```

核心文件：

```text
demo.js   页面渲染和接口调用入口
demo.css  智慧农业页面样式
```

每个页面只是一个壳，例如：

```text
dashboard.html      智能总览
greenhouses.html    大棚管理
devices.html        设备监测
ai.html             LangChain 中枢
knowledge.html      农业知识库
alarms.html         告警工单
```

页面加载后会执行：

```js
fetch('/agriculture/api/xxx')
```

接口成功就用后端数据渲染，接口失败才使用前端模拟数据。

## 后续怎么完善

### 第一步：继续用模拟后端数据

先在 `SmartAgricultureDemoService` 里改数据，确认页面能跟着变化。

### 第二步：加数据库表

建议先建这些表：

```text
agri_greenhouse        大棚表
agri_device            设备表
agri_crop_batch        作物批次表
agri_sensor_data       环境数据表
agri_work_order        告警工单表
agri_knowledge         知识库表
agri_ai_log            AI 调用日志表
```

### 第三步：拆正式后端分层

后续正式写法建议：

```text
controller/agriculture  接口层
service/agriculture     业务层
mapper/agriculture      MyBatis Mapper
domain/agriculture      实体类
resources/mapper/agriculture XML SQL
```

### 第四步：接 LangChain

推荐单独建 Python AI 服务：

```text
ai-service/
  app.py
  chains/
  tools/
  prompts/
  knowledge/
```

调用流程：

```text
前端
  -> Java: /agriculture/api/langchain/analyze
  -> Python: /ai/analyze
  -> LangChain Agent 调 Tool / RAG
  -> Java 返回给前端
```

Java 这边保留 `/agriculture/api/langchain/analyze`，以后把当前模拟返回改成 HTTP 调 Python 服务即可。

## 农事任务为什么改成告警工单

普通“农事任务”容易显得像手工排班，不突出 AI。

现在改成“告警工单”后，逻辑更清楚：

```text
环境数据异常
  -> LangChain Agent 分析原因
  -> RAG 查询农业知识
  -> 生成处理建议
  -> 创建告警工单
  -> 人去执行补水/通风/巡检
  -> 上传反馈
  -> 关闭告警
```

这样能体现智慧农业不是只展示数据，而是从数据到决策再到执行闭环。
